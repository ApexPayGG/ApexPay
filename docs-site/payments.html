Płatności (Autopay BM)

Endpoint POST /api/v1/payments/initiate uruchamia płatność redirect w bramce Autopay BM. Wymaga uwierzytelnionego użytkownika (JWT / sesja). Kwoty są w groszach (integer).

Inicjacja

POST /api/v1/payments/initiate

Body JSON oraz odpowiedź z polem paymentUrl — po prawej.

Przekieruj użytkownika na paymentUrl (GET). Bramka sandbox: https://pay-accept.bm.pl, produkcja: https://pay.bm.pl.

Żądanie

{
  "amount": 5000,
  "currency": "PLN",
  "description": "Doładowanie portfela"
}

Odpowiedź

{
  "status": "success",
  "data": {
    "paymentUrl": "https://pay-accept.bm.pl/...",
    "orderId": "dep:USER_ID:TIMESTAMP"
  }
}

Hash przy inicjacji (strona bramki)

Parametry redirect są podpisane przez ApexPay. Łańcuch do SHA-256 (hex, lowercase): ServiceID|OrderID|Amount|Description|CustomerEmail|SHARED_KEY — zgodnie z konfiguracją środowiska (AUTOPAY_*).

ITN — powiadomienie o płatności

Endpoint wewnętrzny: POST /internal/webhooks/autopay-itn (formularz URL-encoded). Pole transactions zawiera base64(XML).

Weryfikacja integralności: hash SHA-256 (hex, lowercase) z połączenia pól:

ServiceID|OrderID|RemoteID|Amount|Currency|PaymentStatus|SHARED_KEY

Przy poprawnym hashu i statusie sukcesu następuje księgowanie wpłaty; odpowiedź zawsze 200 z treścią XML (CONFIRMED lub komunikat błędu przy niepoprawnym payloadzie).

Przykładowa struktura XML (schemat logiczny)

<?xml version="1.0" encoding="UTF-8"?>
<transaction>
  <ServiceID>YOUR_SERVICE_ID</ServiceID>
  <OrderID>dep:userId:timestamp</OrderID>
  <RemoteID>REMOTE_TRANSACTION_ID</RemoteID>
  <Amount>5000</Amount>
  <Currency>PLN</Currency>
  <PaymentStatus>SUCCESS</PaymentStatus>
</transaction>

Fragment odpowiedzi potwierdzenia (ITN)

<?xml version="1.0" encoding="UTF-8"?>
<confirmation>
  <Status>CONFIRMED</Status>
</confirmation>
Środowisko Wymagane zmienne: AUTOPAY_SERVICE_ID, AUTOPAY_SHARED_KEY, AUTOPAY_GATEWAY_URL, AUTOPAY_RETURN_URL, AUTOPAY_ITN_URL. Szczegóły: docs/API.md w repozytorium.
