Webhooki B2B

Po wybranych operacjach ApexPay zapisuje zdarzenie w kolejce outbox i wysyła POST JSON na skonfigurowany adres (GET/PUT /api/v1/integrations/config — pole webhookUrl i sekret webhookSecret).

Lista zdarzeń

  • charge.succeeded — charge zaksięgowany; payload m.in. id, amount, splits.
  • charge.refunded — zwrot; m.in. chargeId, refundId, coveredBy.
  • payout.created — utworzenie wypłaty.
  • payout.paid / payout.failed — po rozliczeniu admina.
  • dispute.created — spór PSP; dispute.won, dispute.lost (oraz ewent. dispute.accepted) po rozstrzygnięciu.
  • ride.finalized — rozliczenie kursu SAFE TAXI / POST /api/v1/payments/ride-finalize.

Weryfikacja podpisu (HMAC-SHA256)

Nagłówek x-apexpay-signature zawiera wartość szesnastkową (lowercase) digestu HMAC-SHA256 policzonego z dokładnie tego samego surowego ciała żądania UTF-8 co w Content-Type: application/json. Sekret: webhookSecret z konfiguracji integratora.

Implementacja Odczytaj body jako bufor/string przed parsowaniem JSON. Porównaj digest w sposób odporny na timing attacks (np. crypto.timingSafeEqual na buforach bajtów zdekodowanych z hex).

Node.js (Express)

Surowe body, weryfikacja HMAC, potem JSON.parse.

import crypto from "node:crypto";
import express from "express";

const app = express();
const rawBody = express.raw({ type: "application/json" });

app.post("/webhooks/apexpay", rawBody, (req, res) => {
  const secret = process.env.APEXPAY_WEBHOOK_SECRET; // z GET /integrations/config
  const sig = req.get("x-apexpay-signature");
  if (!secret || !sig) {
    res.sendStatus(400);
    return;
  }
  const expected = crypto
    .createHmac("sha256", secret)
    .update(req.body)
    .digest("hex");
  const a = Buffer.from(sig.trim(), "hex");
  const b = Buffer.from(expected, "hex");
  if (a.length !== b.length || !crypto.timingSafeEqual(a, b)) {
    res.sendStatus(401);
    return;
  }
  const event = JSON.parse(req.body.toString("utf8"));
  res.status(200).json({ received: true });
});

Python (Flask)

get_data(cache=False) przed parsowaniem JSON.

import hmac
import hashlib
from flask import Flask, request

app = Flask(__name__)
WEBHOOK_SECRET = b"your_webhook_secret"  # bytes

@app.post("/webhooks/apexpay")
def apexpay():
    raw = request.get_data(cache=False)
    sig = request.headers.get("x-apexpay-signature", "")
    expected = hmac.new(WEBHOOK_SECRET, raw, hashlib.sha256).hexdigest()
    if not hmac.compare_digest(sig.strip().lower(), expected.lower()):
        return "", 401
    event = request.get_json(silent=True)
    return {"received": True}, 200

Polityka ponowień (retry)

Przy błędach HTTP lub timeoutach stosowany jest backoff: 1 min → 5 min → 1 h, maksymalnie 5 prób dostawy. Po wyczerpaniu prób wpis może trafić do kolejki dead-letter (obsługa administracyjna).
