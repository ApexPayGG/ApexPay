Zwroty (refunds)

POST /api/v1/integrations/charges/:chargeId/refunds — zwrot częściowy lub pełny z wybranym modelem ponoszenia kosztu. Wymagany nagłówek Idempotency-Key.

Endpoint

POST /api/v1/integrations/charges/:chargeId/refunds

Przykładowe body JSON — po prawej (coveredBy: PLATFORM, CONNECTED_ACCOUNT lub SPLIT).

Body

{
  "amount": 3000,
  "coveredBy": "SPLIT",
  "reason": "Zwrot reklamacyjny"
}

Trzy modele kosztów (coveredBy)

  • PLATFORM — koszt ponosi portfel użytkownika platformy (zmienna środowiskowa APEXPAY_PLATFORM_USER_ID lub SAFE_TAXI_PLATFORM_USER_ID w zależności od konfiguracji).
  • CONNECTED_ACCOUNT — zwrot z portfeli subkont uczestniczących w oryginalnym splitcie; debet może zejść poniżej zera.
  • SPLIT — podział proporcjonalny do rzeczywistych kredytów z ledgera charge (platforma vs subkonta).

Zwroty częściowe

Wiele żądań zwrotu jest dozwolone, dopóki suma kwot zwrotów w statusie SUCCEEDED nie przekracza kwoty charge. Każde wywołanie powinno mieć unikalny Idempotency-Key.

Okno czasowe

Zwroty są dozwolone wyłącznie w ciągu 180 dni od createdAt charge. Po przekroczeniu — odpowiedź błędu walidacji.

Ledger i webhook

Księgowanie używa typów REFUND_DEBIT / REFUND_CREDIT z referencjami ref:{refundId}:…. Po sukcesie wysyłane jest zdarzenie charge.refunded (patrz Webhooki).
