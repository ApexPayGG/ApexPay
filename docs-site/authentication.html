Uwierzytelnianie

ApexPay obsługuje dwa modele: JWT (sesja użytkownika / panel) oraz klucz API (integracje serwerowe B2B na wybranych trasach).

JWT (panel użytkownika)

Po POST /api/v1/auth/login otrzymujesz token JWT (zwykle zaczyna się od eyJ). Przekazuj go jako:

  • nagłówek Authorization: Bearer <jwt>, lub
  • cookie jwt (httpOnly) w przeglądarce.

JWT służy m.in. do tworzenia kluczy API (POST /api/v1/api-keys), odczytu konfiguracji integratora i list zasobów tam, gdzie dokumentacja dopuszcza „JWT lub klucz”.

Klucz API (integracje B2B)

Na endpointach integratorskich możesz użyć dedykowanego klucza z prefiksem apx_live_. Przekaż go jednym z poniższych sposobów:

  • x-api-key: apx_live_... (zalecane dla serwisów backendowych)
  • Authorization: Bearer apx_live_... — wyłącznie gdy wartość po Bearer zaczyna się od apx_live_ (tokeny JWT nadal używają tego samego nagłówka z prefiksem eyJ).
Bezpieczeństwo Klucz jest hashowany w bazie (bcrypt); surowy sekret znany jest wyłącznie w odpowiedzi na POST /api/v1/api-keys. Nie umieszczaj go w aplikacjach klienckich ani w repozytoriach.

Generowanie klucza w panelu

  1. Zaloguj się przez POST /api/v1/auth/login.
  2. Wywołaj POST /api/v1/api-keys z ciałem { "name": "…" }.
  3. Zapisz pole key z odpowiedzi 201 — nie będzie ponownie wyświetlone.

Przykład żądania z kluczem API

Test połączenia: GET /api/v1/integrations/me zwraca userId i role powiązane z kluczem.

GET /api/v1/integrations/me 
curl -sS "https://api.apexpay.pl/api/v1/integrations/me" \
  -H "x-api-key: apx_live_YOUR_KEY"
{
  "userId": "clxxxxxxxxxxxxxxxx",
  "role": "PLAYER"
}
